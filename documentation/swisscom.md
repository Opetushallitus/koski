# Swisscom-varmenteen päivitys/vaihtaminen

Swisscomin speksi:

The access certificate to be used by the subscriber is a self-signed SSL client certificate generated by the
Applicant directly or on its behalf with a key length consistent with current certificate guidelines
(currently at least 3072 bit for RSA, 256 bit SHA2) for authentication to the AIS Service and is transferred
to Swisscom (Schweiz) AG in advance.

Content of the "Subject" or "Distinguished Name" of the certificate:

    CN = <URL of the subscriber system that implements the communication with AIS>
    emailAddress = <e-mail address of the contact person for this connection>
    O = <Name of the organisation>
    C = <Country of the organisation>

Validity: 3 years. No special requirements are imposed on the key usage.

## Vaihe 1A: Luo/uusi client cert - Let's encrypt

Esim. Let's encryptillä

    % brew install letsencrypt
    % certbot certonly -d pdf-test.koski-luovutuspalvelu-test-certs.testiopintopolku.fi --manual --preferred-challenges dns --config-dir ~/.certbot/config --logs-dir ~/.certbot/logs --work-dir ~/.certbot/work --register-unsafely-without-email --key-type rsa --rsa-key-size 3072

Noudata ohjeita ja tee ACME-autentikointi AWS konsolilla.

## Vaihe 1B: Luo/uusi client cert - DVV

Vaikka tarvetta tehdä client certtiä DVV:n kautta ei ole, halutaan tämä kuitenkin tehdä, koska antaa käytännön kokemuksen
siitä, miten DVV:n kautta saa luotua client certtejä kolmansien osapuolien välillä kommunikointiin. Eli sama case, kuin
se, että Kelan yms. pitää jatkossa käyttää Kosken luovutuspalvelun kanssa DVV:n certtejä, koska me emme muita hyväksy.

Sähköpostiosoitteeksi alla asiantuntija KOSKI-tiimistä.

    mkdir /tmp/certs
    chmod 700 /tmp/certs
    cd /tmp/certs

    openssl req -new -newkey rsa:3072 –nodes -rand /dev/urandom -keyout cert.key -out cert.csr –sha256 -subj '/CN=koski.opintopolku.fi/O=Opetushallitus/C=FI/emailAddress=<TODO_EMAIL>'

## Vaihe 2: tee .p12 -tiedosto

Generoi salasana, jolla avain kryptataan, PASSWORD alla:

    openssl pkcs12 -export \
    -in cert.pem \
    -inkey cert.key \
    -out client.p12 \
    -name "swisscom-test-client-cert" \
    -password pass:PASSWORD


## Vaihe 3: .p12 => Java KeyStore (.jks)

    keytool -importkeystore \
    -srckeystore client.p12 \
    -srcstoretype pkcs12 \
    -srcstorepass PASSWORD \
    -destkeystore client.jks \
    -deststoretype jks \
    -deststorepass PASSWORD

## Vaihe 4: base64-enkoodaa

    base64 -i client.jks | pbcopy

### Vaihe 5: Vie AWS secrets manageriin

Pastea base64-enkoodattu keystore ja valittu salasana swisscom-secrets sisältöön.
